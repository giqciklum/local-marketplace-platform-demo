What this is

A complete, shareable answer to "build vs. buy"

This package is a repeatable, AI-assisted method for sizing an in-house rebuild of any observable SaaS product — applied end-to-end to a Redbox-powered local marketplace. Everything below is original, clean-room work derived from public evidence.

🔎 Observe → Specify

The live site, sitemap, app-store listings and public vendor docs were mined into a 311-feature parity matrix and full PRDs — never the vendor's private code.

📐 Architect

A modern AWS, multi-tenant, event-driven reference architecture — captured as 17 professional draw.io diagrams (HLD, LLD, C4, ERD, sequences).

🧱 Prove it

A working clone: a polished customer storefront and operator console, an Express API, a Postgres schema, Terraform, Docker and CI/CD.

Clean-room boundary. No proprietary Redbox/Korelogic source code, private APIs, database schemas, assets or trademarks were used or copied. Requirements were inferred from publicly observable behaviour and public documentation, and are clearly labelled as such. The result is an independent, functionally-equivalent design — not a copy.
The method

How we turned a public website into a backlog

A six-stage pipeline, each stage accelerated by AI agents and verified by humans. The full playbook is in the methodology document.

Black-box the live site: routes, network, bundle, app listings, public docs.

Catalogue

Size

Validate

The estimate

How many people, and how long?

One launch at full parity — no partial MVP. We don't commit a build timeline or team size up front: a short Discovery firms the real numbers, governed by the bottom-up Estimation Worksheet (112 components).

Delivery model — no MVP

100% parity
  • Single public launch at full feature parity (311 features)
  • Built in internal increments — not a partial customer release
  • Then migrate & cut over live customers from the incumbent

Real numbers come from Discovery

6-week Discovery
  • Locks parity scope, sequencing and dependencies
  • Firms the real timeline, team and cost for Build
  • Outputs a capped, fixed-price Phase 1 proposal

Funded phase by phase

Gated build
  • Go/no-go gate at every phase — stop or steer
  • Tenant-by-tenant cutover — no big-bang
  • Cost is bottom-up via the Estimation Worksheet (112 components)
Story-point roll-ups, team-composition options and phasing are detailed in 40-BACKLOG-ESTIMATION.md, the roadmap, and the bottom-up Estimation Worksheet (112 components). Discovery converts these into a firm, capped Phase 1 proposal.
